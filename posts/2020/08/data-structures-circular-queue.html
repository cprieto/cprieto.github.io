Data structures, a circular queue in C++

13 August 2020 on theory, data structures, c++ 4 minutes reading

Last time we took a look at a simple linear queue with a list using Python, this implementation was super simple and basically we have to care nothing about the size or capacity of the queue. This is fantastic and I am pretty sure most of the time you won’t have to take care of the capacity of the queue.

Do you remember I mentioned there are many ways to implement a simple queue and one of them was using an array or vector?, well, in many languages (like C/C++ or Go) you cannot easily resize a vector and you cannot declare a vector “dynamic” at run time (yes, I am pretty sure some of you will scream: “hey, you can do auto elems = new int[n] in C++ but surprisingly that is not standard and it will fail in some compilers, like MS VC++). This brings some challenges, what do we do when we surpass the capacity of our initial array?

  • We could throw an exception or error and keeps our life moving
  • We could create a new bigger array, move all the data to the new array and continue as nothing happened

The last option sounds pretty fair but it will incur in some serious performance problems and easily your queue will go \(\Theta(n)\) for a common operation like enqueue. Another problem is with dequeue, if we expand our initial array and then consume most of their elements we are “wasting” space with unused content so we have to think about shrinking our queue in some dequeue operations, incurring in another \(\Theta(n)\) problem.

#pragma once
#include <array>

template<typename T, size_t S>
class CircularQueue {
    std::array<T, S> elements{};
    size_t top{0};
    size_t tail{0};
public:
    CircularQueue() = default;
    ~CircularQueue() = default;

    T dequeue();
    T peek() const;
    void enqueue(T element) noexcept;
    [[nodiscard]] bool isEmpty() const noexcept;
};

template<typename T, size_t S>
bool CircularQueue<T, S>::isEmpty() const noexcept {
    return top == tail;
}

template<typename T, size_t S>
void CircularQueue<T, S>::enqueue(T element) noexcept {
    elements.at(tail++) = element;
}

template<typename T, size_t S>
T CircularQueue<T, S>::peek() const {
    if (top == tail) throw std::exception{"Queue is empty"};
    return elements.at(top);
}

template<typename T, size_t S>
T CircularQueue<T, S>::dequeue() {
    if (top == tail) throw std::exception{"Queue is empty"};
    return elements.at(top++);
}

And the unit tests (using Catch2) for our implementation is a little verbose but hey, it is C++ after all!

#define CATCH_CONFIG_MAIN
#include <catch2/catch.hpp>
#include "CircularQueue.h"

TEST_CASE("It is empty", "[circular queue]") {
    CircularQueue<int, 10> queue;
    CHECK(queue.isEmpty());
}

TEST_CASE("Normal enqueue-dequeue", "[circular queue]") {
    CircularQueue<int, 10> queue;
    queue.enqueue(1);
    queue.enqueue(2);
    CHECK_FALSE(queue.isEmpty());
    REQUIRE(queue.dequeue() == 1);
    CHECK_FALSE(queue.isEmpty());
    REQUIRE(queue.dequeue() == 2);
    CHECK(queue.isEmpty());
}

TEST_CASE("Peek doesn't dequeue", "[circular queue]") {
    CircularQueue<int, 10> queue;
    queue.enqueue(1);
    REQUIRE(queue.peek() == 1);
    CHECK_FALSE(queue.isEmpty());
}

TEST_CASE("Dequeue and peek throw if empty", "[circular queue]") {
    CircularQueue<int, 10> queue;
    REQUIRE_THROWS_WITH(queue.dequeue(), "Queue is empty");
    REQUIRE_THROWS_WITH(queue.peek(), "Queue is empty");
}

TEST_CASE("When queue limit is reached, we discard top and move", "[circular queue]") {
    CircularQueue<int, 2> queue;
    queue.enqueue(1);
    queue.enqueue(2);
    queue.enqueue(3);
    REQUIRE(queue.peek() == 2);
}

In the same way as with Python, the standard library in C++ has a really good and powerful implementation for queues using an underline container, as usual, use any of those instead of this sample code!
